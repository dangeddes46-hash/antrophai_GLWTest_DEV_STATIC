# Hosted handoff notes — AntrophAI v0.41.77

## Read this first

The current build is a local single-player GLW prototype. It should not be deployed as real multiplayer by simply publishing the React app.

For hosted play, the server must become authoritative for:

- accounts and identity
- round state
- build/train/explore timing
- combat resolution
- revives and protection
- missiles and Star Wars interception
- LRC firing sequences
- alliance membership and locks
- news/messages generated by combat systems
- debug/admin tools and permissions

The React app should be treated as a working reference UI and gameplay reconstruction, not as the final trust boundary.

## Current gameplay systems in usable prototype state

- GLW launcher and reset flow
- species selection and local player setup
- build/train/science basics
- attacks, revives, land gain and building loss reports
- missile launching, missile impact, blast shields and Star Wars interception
- LRC build/quota/target/fire sequence
- LRC timed shots: first shot immediately, later shots one IG hour / two ticks apart
- LRC alliance locks while firing/being fired upon
- scanner placeholder curve for explore returns
- rankings without ID column
- debug export
- Signal for Assistance page

## Deferred systems

- Spies are deliberately deferred until old-player input confirms how the original mechanic worked.
- Hosted multiplayer state and anti-cheat are not implemented.
- Some archive/library/image manifests contain placeholder paths and should not be treated as deployment blockers.
- IP/public-facing wording remains hybrid/safe by default; Retro Mode is admin/reference only.

## Local-only/admin tools

Admin/prototype tools exist for QA and must not be exposed to normal hosted players without server-side permission checks.

Known tools include:

- add missiles
- add scanners
- add energy
- ready LRC
- launch 100 missiles at the player
- export debug file
- skip time

## Recommended hosted architecture direction

Split the monolithic prototype into server-side rules modules and client UI modules:

- `combatResolver`
- `missiles`
- `lrc`
- `alliances`
- `economyTick`
- `explore`
- `botLogic`
- `debugExport`
- `pages/` UI components

Random game outcomes should be resolved server-side or through deterministic seeded resolution. Do not leave meaningful random outcomes to the client.
